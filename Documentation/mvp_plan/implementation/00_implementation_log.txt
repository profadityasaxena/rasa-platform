SESSION 7 (BUG FIX — TEST LOGIN SYSTEM)
  Goal: Investigate and fix test account login. All 8 accounts were unreachable
        despite correct users/passwords in MongoDB.

  ROOT CAUSE (confirmed via source code trace + curl testing):
    auth.ts had session.strategy = "database"
    Auth.js v5 Credentials provider ALWAYS issues a JWT cookie on login,
    regardless of the configured strategy. The database strategy then tries to
    look up that JWT in the sessions collection as a session token → not found
    → null session → dashboard redirects back to /login. A silent infinite loop.

    Secondary: no jwt() callback → role was never stored in the JWT.
    Secondary: session() callback read from `user` (database pattern) instead of
               `token` (JWT pattern) → role and id were undefined in session.

  VERIFICATION:
    - DB query: all 8 users present, passwords correct, bcrypt verified ✓
    - curl test: POST /api/auth/callback/credentials → 302 to /dashboard + cookie set
    - curl test: GET /api/auth/session with session cookie → {} (empty) BEFORE fix
    - After fix: role=platform_admin, HAS_ID for all 8 accounts ✓

  FIX (auth.ts):
    1. Changed session.strategy from "database" to "jwt"
    2. Added jwt() callback: stores token.id and token.role on sign-in
    3. Changed session() callback: reads from token (not user)
    4. Added declare module "next-auth/jwt" { interface JWT { id, role } }

SESSION 9 (CREDIT FORMULA STANDARDISATION)
  Goal: Remove NGO-controlled credit rates. Standardise to platform-controlled
        formula: credits = floor(actual_minutes_worked). 1 credit per minute.

  DESIGN DECISION:
    NGOs may NOT set credit values. They specify only:
      1. volunteer_capacity  — how many volunteers can attend
      2. estimatedDurationMinutes — expected work duration

    The platform calculates credits from verified attendance timestamps.
    This ensures fairness, economy stability, and manipulation prevention.

  DATA CHANGES:
    lib/db/models/opportunity.model.ts  creditsPerHour → estimatedDurationMinutes
    modules/opportunity/types.ts        Same (DTO, CreateInput, UpdateInput)
    modules/opportunity/validators.ts   creditsPerHour removed, estimatedDurationMinutes added (1–1440)
    modules/opportunity/repository.ts   totalCreditsPool = estimatedDurationMinutes × capacity

  CREDIT CALCULATION CHANGE (checkout):
    modules/participation/service.ts
    BEFORE: credits = floor(actualHours) × opportunity.creditsPerHour
    AFTER:  credits = floor(actualMinutes)   ← 1 credit per minute, no NGO rate

  UI CHANGES:
    app/(dashboard)/opportunity/create/page.tsx  "Credits per hour" → "Estimated duration (minutes)"
                                                  Informational message added explaining platform control
    app/(dashboard)/discovery/page.tsx           Badge: "{X} cr/hr" → "~{X} credits"
    app/(dashboard)/opportunity/page.tsx         creditsPerHour removed, estimatedDurationMinutes displayed
